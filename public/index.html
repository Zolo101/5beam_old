API endpoints

GET/api/all---- Gets all the levels.

GET/api/level/:id---- Gets a specfic level.

GET/api/level/get/:id---- Gets the leveldata from the level specified.

/api/alland/api/level/:iddo not contain the actual leveldata, only the level info. For that, use/api/level/get/:id



/api/*has a ratelimit of 50 requests per minute.

/api/allhas a ratelimit of 20 requests per minute.



POST/api/level/upload---- Uploads a level. Must have the correct format or it will not be accepted.

Structure (as of 21w04a):

PLEASE DONT USE THIS BECAUSE IT WILL CHANGE MASSIVELY IN V4 ALPHA

{
name: string
author: string
date: number
data: number[][]
version: number
}

will be turned into this in the database:

{
id // Database ID
name // Name of levelpack
author // "Guest"
date // Date.now()
data // Levelpack
version // Structure level
}

When you call the API, you will be given an object like this:

{
status // Status response
data // If successful
}

status can be either success, fail, unsure or ratelimit

On success, you are guaranteed some sort of data.

On fail, you wont get any data.

On unsure, i dont know if you will get data. Hopefully you'll never get this.

On ratelimit, you have requested too many times and wont get any data until the cooldown ends.
