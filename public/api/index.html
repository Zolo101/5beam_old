API endpoints:

GET /api/all ---- Gets all the levels.
GET /api/level/:id ---- Gets a specfic level.
GET /api/level/get/:id ---- Gets the leveldata from the level specified.
POST /api/level/upload ---- Uploads a level. Must have the correct format or it will not be accepted.
/api/alland/api/level/:iddo not contain the actual leveldata, only the level info. For that, use/api/level/get/:id



/api/*has a ratelimit of 1000 requests per 10 minutes.

/api/allhas a ratelimit of 20 requests per 10 minutes.


Structure (as of v4 Alpha):

name: string
author: string
description: string
data: string
struct_version: number
level_version: number

When you call the API, you will be given an object like this:

status: "success" | "fail" | "unsure" | "ratelimit"
data: string | null

status can be either success, fail, unsure or ratelimit

success

You are guaranteed data.

fail

You wont get any data.

unsure (unused)

I dont know if you will get data. Hopefully you'll never get this.

ratelimit

You have requested too many times and wont get any data until the cooldown ends.
